# Project Context: Personal Portfolio & Blog Website

## 1. Project Overview
This document serves as context for AI agents assisting with the development, writing, and expansion of my personal website and blog.

## 2. Professional Identity
- **Primary Title:** GCP Solution Architect & AI Engineer
- **Expertise:** Deploying scalable infrastructure on Google Cloud, training enterprise AI models, and architecting robust solutions that bridge machine learning and production engineering.

## 3. Design & UI/UX Preferences
- **Theme:** Dark theme with contrasting cyberpunk purple and lavender accents (`#0a0a0e` background, `purple-500` and `fuchsia-500` gradients).
- **Vibe:** Sleek, modern, terminal-inspired, and highly technical.
-  Clean, bold hero section, prominent display of titles, stats, and a split layout for the profile image.
  -  Minimalist structural layout with a distinct quick-access toolbar with icons in the navigation header for social links (Linkedin, GitHub, Instagram) and search.

## 4. Website Architecture (Sitemap)
The website includes the following primary pages/routes:
1. **Home:** High-impact landing page highlighting expertise, quick stats, and a call-to-action to read the blog.
2. **About:** Detailed professional background, work history, and mission statement.
3. **Projects:** Showcase of key GCP, AI, and architecture projects with descriptions, tech stacks, and links.
4. **Blog:** Technical writing focusing on GCP, AI, LLMs, and System Architecture.
5. **Contact:** Contact form and social links for networking and inquiries.
6. **Quotes:** Curated collection of inspiring, technical, or architectural quotes. (Deferred for post-MVP)
7. **Facts:** Quick statistics, certifications, achievements, and professional trivia. (Deferred for post-MVP)
8. **Talks (Favourite Talks):** Links, slides, and recordings from past speaking engagements or curated talks from others. (Deferred for post-MVP)

## 5. Technical Stack Guidelines
- **Frontend Framework:** React (functional components, hooks) or Next.js for Static Site Generation (SSG).
- **Styling:** Tailwind CSS (utility-first, responsive design).
- **Icons:** Minimalist SVG icons (e.g., Lucide-React).
- **Routing:** Client-side routing or framework-native routing depending on the final implementation.

## 6. Agent Instructions
When assisting with this project, please:
- Maintain the established cyberpunk/dark theme in any new UI components.
- Keep the tone of copy professional, technical, yet accessible.
- Ensure all generated code is responsive and mobile-friendly.
- Follow modern React/Next.js best practices.
